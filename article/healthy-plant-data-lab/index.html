Overview

Electronics are part of our everyday life, but how do they work? Majority of the devices that we rely on for communication, entertainment, & in business are built on programs, circuits, & sensors.

Some questions to think about as you go through this lesson are:

  • What are appropriate sensors to collect the data we need?
  • How do we use these components together with the software?
  • What signals do we want to use to alert our user?

Application

You’ve been tasked to design the HEALTHY PLANT MONITOR, a sophisticated device that helps gardeners ensure the health and growth of potted plants.

We’ve interviewed the top gardeners in Georgia, and they’ve requested the following features:

  • A way to know if a plant is receiving adequate light
  • A signal when a plant is in the dark
  • A numerical reading of how hydrated a plant is
  • A visualization or indicator of when a plant is hydrated

Goals

In this lesson, you’ll be given the opportunity to use the Learn Robotics Workbench to model your Healthy Plant Device.

These are the goals you’ll achieve at the completion of this lesson.

  1. Learn how to communicate & program an electronic controller (Arduino).
  2. Select appropriate sensors to model data
  3. Send data to output devices
  4. Receive data from input devices
  5. Use sensor data to make simple decisions

Lab Activity #1 Using LED’s as Indicators

Using the Information from the Gardener’s requests, we will first want to learn how to turn on an indicator, or any LED in the workbench. When the light is ON, it’ll signal to the gardener that something happened with his/her plant.

  1. Let’s first create an integer variable to store the LED’s connection pin in the code
int red_led = 12; //where the name red_led means the value 12

Activity 1

▶ Define integer variables for all of your LED’s, Potentiometer, button, and LDR. The pin numbers and variable names are given below.

  • red_led = 12
  • yellow_led = 11 ~
  • green_led = 10 ~
  • white_led = 9 ~
  • button = 2
  • pot = A3
  • ldr = A4
  1. In the setup() method, we need to initialize all of the variables as inputs or outputs.
pinMode(red_led, OUTPUT); //where the red_led is an output

Activity 2

▶ Add the pinMode methods for each device in the workbench to your setup() method (between the two curly braces). You’ll need to decide which devices are outputs and which are inputs. Hint: Outputs are sending something to the world, and inputs take data from the world.

  1. To write a value to an LED, we can use the digitalWrite(pin,status) method. Let’s turn the Red LED ON for 1 second, then OFF for 1 second.

Activity 3

▶ Add the following code to your loop() method (between the two curly braces).

	digitalWrite(red_led, HIGH); //turn red LED ON
	delay(1000); //delay for 1 second or 1000 milliseconds
	digitalWrite(red_led, LOW); //turn red LED OFF
	delay(1000); //delay for 1 second or 1000 milliseconds

  1. File > Save. Then upload the code to your board.

  2. If everything is working, you should see your Red LED blink.

Activity 4

▶ Run the same code for all of your LED’s to make sure they’re working.

Lab Activity #2 Methods, Reading, & Displaying Data

  1. Let’s move our code out of the loop() and into a separate method called led_blink().
    A method is a set of procedures that allows us to reuse code for specific functionality.

Activity 5

▶ Copy and paste your code from loop into the new led_blink() method.

void led_blink(int led, int dur){
	 //… paste your code from loop here
}

You can use input parameters to replace red_led and the hardcoded time value if you’d like to reuse this method for every led on the workbench.

void led_blink(int led, int dur){
	//… paste your code from loop here
	digitalWrite(led, HIGH); //turn input LED ON
	delay(dur); //delay for input duration
	digitalWrite(led, LOW); //turn input LED OFF
	delay(dur); //delay for input duration
}

Then when you want to turn on red_led on for 1 second, you can call this method…

led_on(red_led, 1000); //turn red_led on for 1 second

…rather than write those same four lines of code a bunch of times.

  1. In the setup() method, we need to initialize the Serial Monitor & Plotter to accept communication from the Arduino.

Activity 6

▶ Add this line of code inside your setup() method.

Serial.begin(9600); //turn on serial comm @ 9600 baud
  1. Let’s grab readings from the Analog Devices and store them into a reading variable. This command can be used for both the Pot & the LDR.
int pot_reading = analogRead(pot); //read the potentiometer
int ldr_reading = analogRead(ldr); //get the value from the ldr
  1. Next, we’ll print these values to the Serial Monitor.

Activity 7

▶ Add these line of code inside your loop() method.

Serial.print(“Pot value: ”);
Serial.println(pot_reading);
delay(1000); //every second get a new reading
  1. Test out your code!

Activity 8

▶ Verify & Upload the code to your board.
▶ Open up the Serial Monitor. Tools > Serial Monitor. Or click the Magnifying Glass in the top right corner. Hint: Make sure the baud rate is set to 9600 in the bottom right corner of the Serial Monitor Window.
▶ Rotate the pot. ▶ Answer the following question.

What numbers do you see?

Activity 9

▶ Use your cell phone flashlight to see the values change from the LDR.
▶ Open up the Serial Plotter. Tools > Serial Plotter
▶ Change the values of the pot and LDR by rotating the dial or exposing the LDR to light/dark.
▶ Take a screenshot of your real-time plot.
