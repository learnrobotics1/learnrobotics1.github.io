Overview

Robotics combines programming, electronics, and mechanics into a device that helps us perform tasks in the real-world. We will use the affordable Meet Edison Robot to complete interactive robotics labs.

Some questions to think about as you go through this lesson are:

  • How do we develop an educational lesson using robots?
  • What process steps do we need to take to achieve our outcome?
  • How can we modify our lesson to cater to new and advanced learners?

Goals

In this lesson, you’ll be given the opportunity to use the Meet Edison Robot to create solutions to the Lab Activities.

These are the goals you’ll achieve at the completion of this lesson.

  1. Learn how to program the Edison robot using Edware
  2. Understand how to develop a process map
  3. Translate process maps into code
  4. Design lesson plans for Edison that you can use in your class

Download the Software

You’ll need to have the Edware App to program the Meet Edison Robot.

Download the code here: EdWare App for Chrome .

Parts of the Meet Edison Robot

Edison is compatible with LEGO™ Bricks. If you have a box of these in your classroom, it’s a great way to get students learning robotics with a familiar toy!

EdWare - Interactive Development Environment (IDE)

We’re going to write all of our code using EdWare . You can also write programs using Edison’s version of Python. We recommend moving to Python once students are comfortable with typing and programming.

If you’ve ever programmed using function blocks, then EdWare will look familiar to you.

Steps to Download Code to your Robot

Follow these steps to get code onto the Edison:

  1. Power the Edison ON
  2. Plug the EdComm programming cable into the bottom of the robot.
  3. Plug the headphone end of the EdComm programming cable into your computer’s headphone jack.
  4. Turn the Volume Up to the HIGHEST setting on your computer.
  5. Go to the top right corner of the Edware App IDE and click ▶ Program Edison
  6. Click the Circle Button ONCE so that both LEDs are ON (not flashing)
  7. Press the “Program Edison” button on the computer Edware App IDE
  8. You should hear a buzzing noise and then a beep once the program is uploaded.

Running Code on Your Robot

STARTING CODE: Press the triangle button on the robot to start the code.
STOPPING CODE: Press the square button on the robot to stop the code.

Command Blocks to Control Your Robot

  • Control blocks provide functionality to the robot’s sensors & motors
  • Read blocks allow you to receive data from the sensors
  • Data blocks use information to perform calculations
  • Flow blocks help organize the process steps your robot takes

How to Use the Control Blocks

  1. Drag a control block into the Programming Area.
  2. Control blocks are read sequentially LEFT to RIGHT from the Black Start block to the Black End Block.


Lab Activity #1 Robot Dance Party

Goal! Make your robot dance by implementing the following process:

  • Blink Left LED 1 second, then shut off
  • Blink Right LED 1 second, then shut off
  • Beep for ½ second
  • Spin Counter-Clockwise
  • Beep 2X for ½ second
  • Spin Clockwise
  • Repeat

Activity 1

▶ Draw a process map using arrows.

Then…
▶ Translate the process map into code using the Control Blocks in the Edware App.

▶ Test after every other step. It’s difficult to debug your code if there are multiple steps happening. If you test in increments, you can ensure that each step works the way it should!

HINT: Click on a block in the Programming Area to open the Properties Boxes. The left box shows features of the control block. The right box gives a written description of how each block works.

▶ Make modifications as needed

Lab Activity #2 Sumo Wrestling Robots

Goal! Have your robot battle another Edison in the Wrestling Robot Challenge. Here are the requirements for this project:

  • Keep your robot inside the playing field (i.e. stay inside the black lines)
  • Detect the other robot in the playing field, charge after it and try to push it out of the playing field (i.e. obstacle following)
  • Optional Style points - make the robot beep and/or light up before it charges after the other robot

Part A - Stay inside the playing field!

We don’t want Edison leaving the wrestling ring, so we’ll have to program him to detect the boundaries (black lines). We will do this using the Infrared Line Tracker and an IF-Statement. Infrared or IR Sensors work by transmitting a beam of infrared light and receiving the amount of reflected light back. We can use this information to decide whether or not we are on a light or dark surface based on the amount of reflectance.

Combined with an IF statement in our code, we can “check” to see which condition has been met: light surface or dark surface, and move toward or away from it accordingly.

In the EdWare App, you can use the IF loop to check the condition of the line tracker.

Activity 3

▶ Drag an If control block to the Programming Area.
▶ Then click on it to bring up its Properties Box.
▶ Select “Event Happens” > Line Tracker
▶ Choose “On reflective surface” for WHITE/LIGHT surfaces or “On non-reflective surface” for BLACK/DARK surfaces

Start Your Sensors!

Since we are using the line tracker sensor, we need to initialize it at the beginning of our program. We also want to make sure our program runs continuously, so we’ll bring in a loop block. All of our code for this part of the lab will go into the top branch of the loop control block.

Activity 4

▶ Create a process map listing all of the steps you need to account for to keep the robot inside the wrestling ring. The first 3 steps have been listed for you below.

Process Steps Stay inside the ring
1 Drive Forward
2 Detect the line
3 Stop
4 … fill in the next steps!

Activity 5

▶ Translate your process map into robot code. Be sure to test it out after every couple control blocks to make sure it works. This program is longer, so it’ll be hard to debug once the whole code has been written.

Part B - Detect & CHARGE!

We’re going to write a second program to see an object and charge after it. This will simulate the robots “wrestling” as they try to push each other out of the ring. Edison has built-in Infrared (IR) Distance Sensors used to detect if an object is present. Similarly to the IR Line Tracker, we will use the left and right IR Distance Sensors to determine where the opponent is and then move towards it.

Activity 6

▶ Use an IF Statement to check to see if the opponent is on the LEFT side of the robot. If so, you’ll want to move your robot to the LEFT to charge after him!
▶ Do the same for the right side.
▶ Make sure you initialize the Detect Object control block before the loop command.
▶ Create a process map listing all of the steps you need to do to detect and charge after another robot. The first 3 steps have been listed for you below.

Process Steps Charge when you see a robot
1 Drive Forward
2 Detect the robot
3 Drive Forward FAST!
4 … fill in the next steps!

Activity 7

▶ Translate your process map into robot code. Be sure to test it out after every couple control blocks to make sure it works.
▶ Combine your Detect & Charge Code with your code from Part A.
▶ Test out this new version and make sure that both features work. Make modifications as necessary.
